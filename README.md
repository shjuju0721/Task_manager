# 📋 TaskFlow — 개인 작업 관리 앱

> 칸반 보드, 우선순위, 마감일 같은 **프로젝트 관리 기법**으로 내 할 일을 체계적으로 관리하고,
> **대시보드**에서 진행 상황을 한눈에 보는 웹 앱입니다.

회원가입이나 서버 설정이 필요 없습니다. 데이터는 **내 브라우저에 저장**되므로, 받아서 바로 실행하면 됩니다.

```
┌─ 사이드바 ─┬──────────── 대시보드 ────────────┐
│ + 새 작업  │  전체 12   완료율 33%   임박 3   지연 2 │
│            │  ┌──────┐ ┌──────┐ ┌──────────┐      │
│ 대시보드   │  │상태분포│ │우선순위│ │ 7일 활동 │      │
│ 칸반 보드  │  └──────┘ └──────┘ └──────────┘      │
│ 작업 목록  │  오늘 집중할 작업      프로젝트 진행률  │
│ 프로젝트   │  ▸ 대시보드 차트…     포트폴리오 ▓▓░ 66%│
└────────────┴──────────────────────────────────────┘
```

---

## 🚀 시작하기

### 1. 준비물

- **Node.js 18 이상** ([nodejs.org](https://nodejs.org)에서 설치)
- **pnpm** (없다면 한 번만: `npm install -g pnpm`)

### 2. 실행

```bash
cd next-app
pnpm install      # 처음 한 번만 — 필요한 패키지 설치
pnpm dev          # 개발 서버 시작
```

터미널에 `http://localhost:3000` 이 표시되면, 브라우저에서 그 주소를 열면 됩니다. 🎉

> **그 외 명령어**
> - `pnpm build` — 배포용으로 빌드
> - `pnpm typecheck` — 타입 검사
> - `pnpm lint` — 코드 검사

## ✨ 주요 기능 (적용한 프로젝트 관리 기법)

| 기능 | 설명 |
| --- | --- |
| 🗂️ **칸반 보드** | `할 일 → 진행 중 → 완료` 3단계로 작업을 진행. 카드를 **끌어다 놓아** 상태를 바꿉니다. |
| 🔥 **우선순위** | 긴급 · 높음 · 보통 · 낮음 4단계. 색깔로 한눈에 구분됩니다. |
| ⏰ **마감일 관리** | 마감이 **3일 이내**면 "임박", 이미 지났으면 "지연"으로 자동 표시·경고합니다. |
| 📁 **프로젝트별 묶기** | 관련 작업을 프로젝트로 묶고, 프로젝트마다 진행률을 추적합니다. |
| 🏷️ **태그 · 예상 시간** | 작업을 태그로 분류하고 예상 소요 시간을 기록합니다. |
| 🎯 **오늘 집중할 작업** | 우선순위와 마감 임박도를 점수로 계산해, 지금 할 일을 **자동으로 추천**합니다. |

## 🖥️ 화면 구성

| 메뉴 | 하는 일 |
| --- | --- |
| **대시보드** (`/`) | 핵심 지표(전체·완료율·임박·지연), 상태/우선순위 분포 차트, 최근 7일 활동, 프로젝트 진행률, 지연 작업 경고를 한 화면에서 봅니다. |
| **칸반 보드** (`/board`) | 카드를 드래그해 상태를 바꾸고, 프로젝트별로 걸러 봅니다. |
| **작업 목록** (`/tasks`) | 검색하고, 상태·우선순위·프로젝트로 거르고, 정렬해서 봅니다. |
| **프로젝트** (`/projects`) | 프로젝트를 만들고 수정하며, 진행률을 카드로 확인합니다. |

작업 추가/수정은 어느 화면에서나 **`+ 새 작업`** 버튼 또는 카드 클릭으로 할 수 있습니다.
오른쪽 아래(사이드바 하단)에서 **다크 모드**도 켤 수 있어요. (단축키 `d`)

## 💾 내 데이터는 어디에 저장되나요?

모든 작업·프로젝트는 **내 브라우저의 localStorage**(`taskflow.state.v1` 키)에 저장됩니다.

- 새로고침하거나 브라우저를 껐다 켜도 그대로 남아 있습니다.
- 같은 브라우저에서만 보이며, 외부 서버로 전송되지 않습니다.
- 처음 실행하면 둘러보기 좋게 **예시 데이터**가 채워져 있습니다. **프로젝트 화면의 "예시 초기화"** 버튼으로 언제든 처음 상태로 되돌릴 수 있습니다.

## 🛠️ 문제 해결

**`pnpm dev`를 실행했는데 페이지가 안 떠요 / "Another next dev server is already running" 메시지가 나와요.**

이전에 켠 개발 서버가 백그라운드에 남아 충돌하는 경우입니다. 아래로 정리한 뒤 다시 `pnpm dev` 하세요.

```powershell
# Windows PowerShell
Get-CimInstance Win32_Process -Filter "Name='node.exe'" |
  Where-Object { $_.CommandLine -match "next" } |
  ForEach-Object { Stop-Process -Id $_.ProcessId -Force }
```

```bash
# macOS / Linux
pkill -f "next dev"
```

**포트 3000이 이미 쓰이고 있어요.** → 다른 포트로 실행: `pnpm dev -- -p 3001`
